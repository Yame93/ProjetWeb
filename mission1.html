Clés Primaires et Étrangères

Clés Primaires

  • Table Fournisseur = N° fournisseur
  • Table Produit = Réf produit
  • Table Catégorie = Code catégorie
  • Table Clientèle = Code client
  • Table Commande = N° commande
  • Table DetailCommande = Réf produit + N° commande

Clés Étrangères

  • Table Fournisseur = NULL
  • Table Produit = N° fournisseur, Code catégorie
  • Table Catégorie = NULL
  • Table Clientèle = NULL
  • Table Commande = Code client
  • Table DetailCommande = NULL

Problèmes Observés

Dans les villes de la table Fournisseur, il y a « São Paulo » et « Sao Paulo », dans les pays de la même table, il y a « Etats-Unis » et « USA », il manque également un « (e) » à un des « Représentant(e) », il y a des espaces en trop à certains endroits et des majuscules qui manquent. Ces fautes d’orthographes/syntaxes peuvent gêner le triage et filtrage de cette table.

Dans les contacts de la table Fournisseur, il n’y a que cinq « directeur marketing » alors qu’il est sensé en avoir six, il manque aussi un « chef de ventes » et un « directeur marketing international ». Ces trous dans la table peuvent faire perdre beaucoup de temps.

Idées d’Amélioration

  • Mettre en place des restrictions sur les noms pour éviter les doublons et fautes d’orthographes.
  • Séparer davantage les données pour plus de lisibilité.
  • Utiliser un logiciel SQL comme Access à la place d’Excel.
